✗ PROBLEM 1: TAB NAVIGATION IS CROWDED
─────────────────────────────────────────
Current state:
  • 6 tabs crammed into one horizontal row
  • Tabs wrap on mobile/small screens
  • Text can be clipped on narrow displays
  • [Logistics] [Barracks] [Conquest] [Espionage] [Fabrials] [Arena]

Issues:
  • Hard to scan all options at once
  • Hidden tabs require scrolling or guessing
  • No visual hierarchy (all tabs same size/color)
  • Mobile experience is degraded

Recommendation:
  Option A (Grouped Navigation):
    • Create main categories: [MILITARY] [EXPANSION] [INTEL] [MYSTIC] [COMBAT]
    • Sub-tabs appear in secondary row or dropdown
    • Example: MILITARY dropdown → [Barracks] [Conquest]
  
  Option B (Collapse Secondary Tabs):
    • Show only: Logistics, Barracks, Arena
    • Move Conquest/Espionage/Fabrials to modals accessible from main buttons
    • Example: In Logistics, add button "OPEN CONQUEST PANEL" → opens modal
  
  Option C (Vertical Tab Bar - Change Layout):
    • Move tabs to left sidebar (vertical list instead of horizontal)
    • Allows 6+ tabs without wrapping
    • Takes more horizontal space but cleaner
  
  Recommendation: Try Option A (grouped navigation) for balance between
  clarity and screen real estate efficiency.

✗ PROBLEM 2: MODAL EXPLOSION (TOO MANY MODALS)
──────────────────────────────────────────────
Current: 14+ modals in the system
  • Each modal has own styling, scroll behavior, button placement
  • Players must juggle between modals (modal within modal is rare but can happen)
  • Modal hierarchy unclear: Is "Mission Details" modal different tier than "Deploy" modal?

Issues:
  • Hard-coded modal positioning can cause overlaps/awkwardness
  • Mobile: Large modals don't fit screen, forced to scroll inside scrollable area
  • No modal "stacking" - can't easily compare two open modals side-by-side
  • Modal close (X button) is small target on mobile

Recommendations:
  1. Consolidate similar modals:
     - Combine all result modals (Scout/Attack/Plateau/Conquest/Espionage) 
       into single "Mission Result" modal with type-specific sections (use CSS 
       display: none to hide irrelevant sections)
     - Combine all building purchase modals (blue/green/purple/red) into single 
       "Purchase" modal with tabbed interface
  
  2. Add modal sizing options:
     - Small modals (33% width) for quick dialogs
     - Medium modals (50% width) for standard interactions
     - Large modals (80-90% width) for complex ones (Spanreed, Fabrials)
     - Add CSS variable: --modal-width for easy theming
  
  3. Improve mobile modal experience:
     - Full-screen modals on mobile (width: 100vw, height: 100vh)
     - Increase X button size for touch targets (min 44px × 44px)
     - Add swipe-down-to-close gesture support
     - Reduce padding/margins on mobile to fit more content
  
  4. Add modal history/nesting:
     - Players can open modals from inside modals
     - Tracks breadcrumb: Main → Deploy Modal → Mission Details Modal
     - Back button to return to previous modal (without data loss)

✗ PROBLEM 3: INFORMATION DENSITY IMBALANCE
───────────────────────────────────────────
Current state:
  LEFT COLUMN: Very dense (many tabs, lots of lists)
  CENTER COLUMN: Medium density (command hub + game log)
  RIGHT COLUMN: Very sparse (just Spanreed button + message area)

Issues:
  • Left column feels cramped even on large screens
  • Right column is underutilized
  • Small 3-column grid doesn't match content needs
  • On tablets: Right column vanishes entirely, forcing full left-to-center workflow

Recommendation:
  Option A: Adaptive grid
    • Desktop (1920px+): 4 columns [TAB CONTENT (1.5)] [COMMAND HUB (2)] [SPANREED (1.5)]
    • Tablet (1024px): 3 columns [TABS (1)] [COMMAND HUB (2)] [SPANREED (1)] → hide if needed
    • Mobile (480px): 1 column full-width, tabs become row at top
  
  Option B: Rebalance content
    • Move some left-column details to right column
    • Example: Military composition (LEFT) → right side shows power/casualties/upkeep 
    • Center is cleaner, peripheral details on edges
  
  Compression: Consider moving Military Details panel (currently left) to 
  collapsible sidebar that slides in from right (frees up left space).

✗ PROBLEM 4: VISUAL HIERARCHY IS UNCLEAR
──────────────────────────────────────────
Current state:
  • All information appears at similar "importance level"
  • Main title, resource display, tabs, modals all have different visual weights
  • User must scan screen to find what they're looking for
  • No clear "call to action" (what should I do NOW?)

Issues:
  • "COMMAND HUB - PALACE" title is small, easy to miss
  • Mission deployment buttons blend into surrounding panels
  • Resource display is prominent but could obscure critical info during alerts
  • Active mission status is not visually distinct from available missions

Recommendations:
  1. Create visual hierarchy with size/weight:
     • H1: "COMMAND HUB - PALACE" (make larger, bolder)
     • H2: Tab titles within panels (medium size, colored)
     • H3: Mission names, building names (smaller)
     • Body: Stats, numbers, descriptions
  
  2. Highlight active deployments:
     • Active missions get glow/animation effect
     • Different background color or border treatment
     • Example: Active missions have golden border + subtle pulsing glow
  
  3. Create action zones (high-priority areas):
     • "NEXT ACTION" area: Always visible, shows most urgent task
     • Example: "Deployment completes in 2 HOURS → [VIEW MISSION]" button
     • Example: "Spy Network built! → [OPEN ESPIONAGE]" button
     • Example: "Highstorm incoming! → [PREPARE DEFENSES]" banner
  
  4. Use whitespace strategically:
     • Separate distinct sections with visible gaps
     • Reduce padding/margin on less important items
     • Increase padding around critical controls (mission launch buttons)

✗ PROBLEM 5: COLOR CONTRAST & READABILITY
────────────────────────────────────────────
Current state:
  • Uses slate-900/cyan/green/purple/blue color scheme
  • Text colors: white (main), slate-300 (secondary), slate-400 (tertiary)
  • On dark background, some combinations may have low contrast

Issues (Potential - depends on exact hex values):
  • Slate-400 text on slate-900 background may fail WCAG AA contrast standards
  • Cyan text on dark slate may strain eyes in low-light gaming
  • Purple fabrials modal might have readability issues
  • No color-blind mode (can't distinguish red/green easily if colorblind)

Recommendations:
  1. Audit contrast using WCAG Color Contrast Checker:
     • All text should be 4.5:1 ratio minimum (AA standard)
     • Large headings can be 3:1 minimum
     • Test on actual monitor (not just screenshot)
  
  2. Add readability improvements:
     • Increase font-weight on secondary text (slate-400)
     • Add subtle text-shadow on colored backgrounds: 
       "text-shadow: 0 1px 3px rgba(0,0,0,0.8)"
     • Increase line-height on long paragraphs (helps dyslexia-friendly)
  
  3. Add colorblind mode:
     • Add pattern/texture overlay on color-coded buttons
     • Example: Blue + horizontal stripes, Green + dots, Purple + cross-hatch
     • Allows both colorblind and color-visible users to distinguish
     • CSS filter or SVG pattern on background

✗ PROBLEM 6: RESPONSIVE DESIGN FRAGMENTARY
────────────────────────────────────────────
Current state:
  • Uses Tailwind CSS (responsive-first)
  • Breakpoints: lg:col-span-4, md:flex-row
  • Mobile view likely works but may have awkward wrapping

Issues:
  • 3-column grid doesn't gracefully degrade to mobile
  • Modals likely don't scale properly on small screens
  • Mission list might be hard to read on phone
  • Avatar/champion might disappear on small screens entirely
  • Tabs might become impossible to click on mobile (crowded row)

Recommendations:
  1. Establish mobile-first breakpoints:
     • Mobile: < 640px (1 column, full-width panels)
     • Tablet: 640px - 1024px (2 columns)
     • Desktop: > 1024px (3+ columns)
  
  2. Adjust UI density per breakpoint:
     • Mobile: Hide non-essential columns, collapse panels
     • Tablet: Show 2 main panels side-by-side
     • Desktop: Full 3-column layout
  
  3. Make interactive targets mobile-friendly:
     • Buttons: minimum 44px × 44px (mobile standard)
     • Form inputs: 48px height minimum
     • Close (X) button: 40px target size
     • Dropdowns: 48px height to tap easily
  
  4. Test on actual devices:
     • Test on iPhone SE (375px width)
     • Test on iPad (768px width)
     • Test on desktop (1920px width)
     • Use Chrome DevTools Device Emulation during development
  
  Example responsive CSS pattern:
  ```css
  /* Mobile first */
  .main-grid { grid-template-columns: 1fr; }
  
  /* Tablet */
  @media (min-width: 768px) {
    .main-grid { grid-template-columns: 1fr 1fr; }
  }
  
  /* Desktop */
  @media (min-width: 1024px) {
    .main-grid { grid-template-columns: 1fr 2fr 1.5fr; }
  }
  ```

✗ PROBLEM 7: MISSION DEPLOYMENT UX FRICTION
─────────────────────────────────────────────
Current workflow:
  1. User clicks mission button (Scout/Attack/Conquest/etc.)
  2. Deploy modal opens with unit composition sliders
  3. User must manually enter unit counts or use default
  4. User clicks "DEPLOY ARMY" button
  5. Mission details modal opens
  6. User clicks "VIEW MISSION" or waits

Issues:
  • Slider controls for units are hard to use precisely
  • No "quick deploy" option for repeated missions (same units as before)
  • No unit presets (e.g., "Balanced", "Max Power", "Max Carry")
  • Users can't easily compare unit effectiveness before deploying
  • Mission cost (if any) not shown until deployment confirmation

Recommendations:
  1. Add preset army compositions:
     • "BALANCED" preset (mix of units for standard missions)
     • "OFFENSIVE" preset (max power units)
     • "ECONOMIC" preset (cheap units only)
     • "CUSTOM" preset (save last used composition)
     • Each preset is one-click to load + one-click to deploy
  
  2. Replace sliders with number inputs:
     • Show current number in input field
     • Users can type exact number or use ±buttons
     • Much more precise than dragging sliders
     • Example: [−] [50] [+] for 50 Spearmen
  
  3. Add deployment preview:
     • Show total power, casualties estimate, mission success chance
     • Calculate and display before modal opens
     • Example: "115 units | ~850 power | 85% success | 200 spheres cost"
  
  4. Add "Quick Deploy" button for repeated missions:
     • After first deploy, show "QUICK DEPLOY (same units)" button
     • One-click redeploy saves same unit composition
     • Especially useful for farming scout missions

✗ PROBLEM 8: ESPIONAGE INTERFACE COMPLEXITY
─────────────────────────────────────────────
Current state:
  • Espionage has separate modal
  • Shows: recruitment, missions, suspicion, counter-intel
  • Uses select dropdown for target choice
  • No clear workflow (what order should I do these in?)

Issues:
  • New players won't understand espionage mechanics from UI alone
  • Suspicion/counter-intel system not visually explained
  • No tooltips to explain terms like "Suspicion" or "Counter Intelligence"
  • Mission planning modal is separate, adding cognitive load

Recommendations:
  1. Add tooltips/help text:
     • Hovering over "Suspicion" shows: "How likely your spies are discovered (0-100%)"
     • Hovering over "Counter Intelligence" shows: "Info gathered about rival threats"
     • Put question mark icons (?) next to confusing terms
  
  2. Create espionage workflow guide:
     • Show recommended order: (1) Recruit agents (2) Gather intel (3) Plan sabotage
     • Add step numbers or flow arrows showing progression
     • Disable unavailable options with clear reason why
  
  3. Visualize espionage network:
     • Show current agents as cards/icons (not just numbers)
     • Display target rival's defenses/counter-intel as bar chart
     • Compare YOUR agents vs TARGET's defenses visually
  
  4. Simplify target selection:
     • Replace dropdown with rival cards showing portraits/stats
     • Each card shows: name, intel level, suspicion against us, defenses
     • Click card to select target, mission options auto-populate
  
  5. Add espionage quests/tutorial:
     • "First, recruit 2 Nobles for basic missions"
     • "Next, send them to gather intel (safe, low-reward)"
     • "Finally, plan a sabotage operation (risky, high-reward)"
     • Checkmark each stage so new players understand progression

───────────────────────────────────────────────────────────
Current state:
  • Shows champion level, HP, Thrill
  • Shows "Find Opponent" button
  • Tournament card appears conditionally
  • No stat breakdowns or progression graph

Issues:
  • Players don't know how close they are to level-up
  • No way to see past 10 duels or compare duel history
  • Arena fabrials don't show effectiveness (does Half-Shard actually help?)
  • Tournament schedule is unclear (when/how often do tournaments run?)

Recommendations:
  1. Add XP bar visualization:
     • Show progress toward next level (optional)
     • Example: [=====>    ] 45% to Level 3
     • Color changes as progress increases (green for close)
  
  2. Add duel history tab:
     • Show last 10-20 duels with opponent names
     • Clicking duel shows: matchup, result, XP gain, damage taken, fabrials used
     • Trend tracking: win/loss ratio, average damage, arena progression graph
  
  3. Make tournament schedule visible:
     • Add calendar showing tournament dates
     • Example: "Tournament in 3 days [ENTRY CLOSED]" or "[REGISTER NOW]"
     • Show current tournament standings if applicable
  
  4. Show arena fabrial effectiveness:
     • After using Half-Shard, show: "Damage reduction: 42%"
     • After using Thrill Amp, show: "Thrill bonus: +2 per win"
     • Make impact visible to justify Gemheart cost

✗ PROBLEM 10: HIGHSTORM NOTIFICATION FATIGUE
──────────────────────────────────────────────
Current state:
  • 3-phase notification system (overlay → banner → small banner)
  • Appears as fixed element on top of screen
  • Can persist for 10+ minutes with time remaining visible

Issues:
  • Fixed overlay blocks all gameplay for 10 seconds (testing killer)
  • Phase 1 is jarring (want to turn off immediately)
  • Phase 2/3 banners take up screen real estate permanently
  • Click detection on small banner might be hard on mobile
  • No "dismiss" option if player wants to ignore storm notifications

Recommendations:
  1. Reduce Phase 1 overlay duration:
     • Currently 10 seconds, reduce to 3-5 seconds
     • Or make it dismissible (click anywhere to close)
     • In dev mode, maybe reduce to 1 second to speed testing
  
  2. Collapse highstorm banners when not focused:
     • Phase 2/3 banners collapse to single-line compact view
     • Expand on hover to show full "Storm active | Click for details"
     • Prevents permanent real estate loss
  
  3. Add storm prediction:
     • Show "Highstorm arriving in 2 hours" warning before it hits
     • Gives players time to prepare (recall missions, move units, build shelters)
     • Reduces surprise/annoyance factor
  
  4. Make notifications optional:
     • Settings menu: "Highstorm alerts" toggle
     • Players can still see storms in game log/effects, just no banner
     • Reduces notification fatigue without hiding information
  
  5. Add desktop notifications (with permission):
     • Use browser Notification API for OS-level alerts
     • "Highstorm incoming" appears in Windows notification center
     • Even if game minimized, player gets alert
     • More non-intrusive than in-game overlay

SECTION 9: CURRENT UI PAIN POINTS TO FIX FIRST
================================================================================

If you can only fix 3 things, prioritize these:

1. [HIGHEST PRIORITY] Increase button sizes (15 mins)
   → Improves mobile playability immediately

2. [HIGH PRIORITY] Add tooltips (2 hours)
