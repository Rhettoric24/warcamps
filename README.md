# Warcamps Online - A Real-Time Strategy Game

## Vision

**Warcamps Online** is a multiplayer real-time strategy game inspired by Brandon Sanderson's *The Stormlight Archive*. Players assume the role of Highprinces commanding military forces, managing economies, conducting espionage, and participating in dangerous plateau runs. The game uses a **real-time clock mechanic** where 1 real-world hour = 1 in-game day, creating a persistent world that progresses even when you're offline.

### Why This Matters
- **Asymmetric Gameplay**: Military strength, economic dominance, espionage expertise, and combat prowess all create different win paths
- **Persistent World**: Your decisions affect others; offline time grants resource generation
- **Thematic Alignment**: Captures the Stormlight aesthetic of warcamps, shardblades, and chasmfiends
- **Emergent Stories**: Player interactions (spying, alliances, raids) create unique narratives

## Core Mechanics: Time & Progression

### Time System
- **1 Real Hour = 1 In-Game Day**
- **1 In-Game Day = 24 Hours** (split into dawn, highstorms, etc. conceptually)
- **1 In-Game Month = 24 Days**
- **1 In-Game Year = 168 Days** (7 months)

**Example Timeline:**
| Real Time | In-Game | Events |
|-----------|---------|--------|
| 1 hour | 1 day | Resources generate, buildings complete, troops garrison |
| 8 hours | 8 days | Market income accrues, espionage missions resolve |
| 1 week | 7 weeks | Major construction completed, alliances formed/broken |

### Processing Offline Time
When you log back in, the game calculates elapsed real time and simulates **all daily ticks** (resource generation, building completion, etc.) instantly. This prevents players from being penalized for offline time. Including a server to keep the game running completely while some players are offline would be the end goal.

## Game Systems

### 1. **Military System**
Command armies composed of multiple unit types, each with distinct roles and stats.

#### Unit Types
| Unit | Cost | Provisions | Power | Specialization |
|------|------|-----------|-------|-----------------|
| **Bridgecrew** | 5 Spheres | 1 | 0.1 | Cheap filler; early-game recruiting |
| **Spearmen** | 10 Spheres | 2 | 1.0 | Standard melee infantry |
| **Archers** | 15 Spheres | 2 | 1.0 | Ranged damage output |
| **Chulls** | 50 Spheres | 5 | 0.5 | Transportation (carry capacity) |
| **Shardbearers** | 1 Gemheart | 3 | 2.0x multiplier | Elite warriors; multiply army power |
| **Noble Spies** | 50 Spheres | 2 | — | Espionage agent (spy power: 1) |
| **Spies** | 150 Spheres | 3 | — | Professional spies (spy power: 2) |
| **Ghostbloods** | 600 Spheres | 5 | — | Elite assassins (spy power: 5) |

#### Military Stats
- **Power**: Determines combat effectiveness in plateau runs
- **Carry**: Cargo capacity for provisions; enables missions requiring supply transport
- **Speed**: Affects deployment speed and mission completion time
- **Survival**: Percentage chance units survive missions without casualty
- **Population**: Total unit count; limited by provisions cap

#### Army Composition Example
- **Early Game (Cheap Military)**: 100 Spearmen + 50 Archers = 150 power, low cost
- **Mid Game (Economic Boost)**: +Training Camp building → military power multiplier
- **Late Game (Shardbearers)**: 3 Shardbearers + 50 Spearmen = 350+ power, exponential scaling

### 2. **Economic System**
Resources drive all actions: recruiting troops, building structures, and conducting espionage.

#### Resource Types
- **Spheres** (Currency)
  - Starting: 15,000 (Dev Mode: 25,000)
  - Generated by: Markets, Gemheart ownership, raid loot
  - Used for: Recruitment, buildings, spy operations
  
- **Gemhearts** (Premium Resource)
  - Starting: 0 (Dev Mode: 10)
  - Generated by: Winning plateau runs, black market purchases (10,000 Spheres each)
  - Used for: Shardbearers, Fabrials, economy acceleration

#### Income Sources
| Source | Rate | Requirement | Notes |
|--------|------|-------------|-------|
| Market | +100 S/day | Build Market building | Exponential cost; stackable |
| Gemheart Ownership | +20 S/day | Own Gemhearts | Passive income |
| Plateau Run Victory | +Variable | Win run as fastest | Split prize pool by power×carry |
| Ledger Fabrial | ×1.5 multiplier | 5 Gemhearts | Boosts Market income |

#### Building System

**Production Buildings** (Generate Resources)
- **Market**: Costs 250 S (exponential scaling). Generates +100 S/day. No cap.

**Military Buildings** (Enhance Army)
- **Training Camp**: Costs 7,000 S (linear: +5,000 each). Grants +10% army power per building.
- **Monastery**: Costs 7,000 S (linear). Grants +5% troop survival rate per building.

**Strategic Buildings** (Unlock Systems)
- **Spy Network**: Costs 5,000 S (flat). **Required** to conduct espionage.
- **Research Library**: Costs 10,000 S (flat). **Required** to construct Fabrials.
- **Whisper Tower**: Costs 10 Ghostbloods (special). Enhances spy power by 1.5x.
- **Soulcast Shelter**: Costs 100 S (exponential). Provides provisions storage; cap 10.

#### Progression Path (Example)
1. **Day 1**: Start with 15,000 Spheres, 100 Spearmen + 20 Bridgecrews
2. **Days 1-2**: Build 5-10 Markets → +500-1000S/day passive income
3. **Day 3-4**: Build Spy Network (5,000 S) → Unlock espionage
4. **Day 5+**: Research Library (10,000 S) → Access Fabrials
5. **Week 2+**: Fabrials create exponential income/power scaling

### 3. **Plateau Run System**
Random events where Chasmfiends spawn and NPCs form coalitions to hunt them. **Central gameplay loop**.

#### Event Trigger
- Chasmfiends spawn randomly (approximately every few hours of gameplay)
- Warning phase: 30 seconds to muster forces
- Alert sent to all players with active notifications

#### Mechanics
1. **Coalition Formation**
   - Player optionally commits units to the run
   - NPCs automatically join with their available forces
   - Participants sorted by initial "speed" (based on unit composition)

2. **Run Resolution**
   - **Victory Condition**: Total allied power ≥ Chasmfiend power
   - **Defeat Condition**: Total allied power < Chasmfiend power
   - Power calculated as: `units × power × training_bonus × shardbearers_multiplier`

3. **Rewards (Victory Only)**
   - **Gemheart Distribution**: 
     - Fastest coalition member (highest speed) wins 1 Gemheart
     - If player is fastest: Gemheart + 5,000 Sphere finder's fee
     - If player is not fastest: Proportional share of 50,000-sphere consolation pool (based on power × carry weight)
   
   - **Alternative Victory**: No gemheart but high-power, high-carry army → 25,000+ Spheres

#### NPC Lords (Allies/Rivals)
| Lord | Agents | Spheres | Gemhearts | Shardbearers | Arena Lvl | Notes |
|------|--------|---------|-----------|--------------|-----------|-------|
| **Sadeas** | 15 | 15,000 | 2 | 2 | 5 | Aggressive; frequently joins runs |
| **Taravangian** | 50 | 5,000 | 10 | 0 | 2 | Spy-focused; good intelligence |
| **Sebarial** | 10 | 80,000 | 5 | 0 | 1 | Wealthy but weak military |
| **Dalinar** | 20 | 5,000 | 0 | 1 | 6 | Honorable; powerful shardbearer |

### 4. **Espionage System**
Gather intelligence on rivals, conduct daring heists, and sabotage enemies.

#### Spy Resources
Three spy types with increasing power and cost:
- **Nobles**: 1 spy power, cheapest
- **Spies**: 2 spy power, medium cost
- **Ghostbloods**: 5 spy power, expensive (also build Whisper Tower)

#### Espionage Actions

**Reconnaissance Operations**
- **Scout** (1 Spy Power): Gather basic info on target's military/buildings; low suspicion
- **Spy** (2 Spy Power): Extract detailed unit counts and wealth; medium suspicion
- **Infiltrate** (4 Spy Power): Deep intelligence; high suspicion

**Active Operations**
- **Heist** (3 Spy Power): Steal 5,000-15,000 Spheres; high suspicion
- **Sabotage** (5 Spy Power): Damage building or army; very high suspicion
- **Assassinate** (8 Spy Power): Eliminate units or agents; extreme suspicion

#### Suspicion System
Each rival tracks your threat level on a 0-100 scale:

| Threshold | Level | Effect |
|-----------|-------|--------|
| 0-24 | **Unknown** | 100% action success rate |
| 25-59 | **Known** | 70% action success rate; counter-intel active |
| 60-89 | **Suspicious** | 40% success rate; heavy counter-measures |
| 90-100 | **Hostile** | 10% success rate; likely discovery and retaliation |

**Suspicion Decay**:
- Decays exponentially if you stop spying on a target
- Day 1 without spying: -1 suspicion
- Day 2 without spying: -2 suspicion
- Day 3 without spying: -4 suspicion
- Encourages rotation of targets rather than focusing one rival

#### Reports & Intelligence
- Successful operations generate intel reports in an in-game journal
- Read detailed target profiles: unit counts, sphere wealth, building status
- Plan coordinated attacks with other players using shared intelligence

---

### 5. **Arena Combat System**
Personal duels between champion warriors; level up and test your mettle.

#### Duel Mechanics
Each player commands a **Champion** with stats:
- **Level**: Ranges 1-∞ (exponential XP scaling)
- **HP**: 3 per level (max 30 at level 10)
- **Thrill Pool**: 10 thrill points per duel (refreshes daily)

#### How Dueling Works
1. **Bid Thrill**: Both combatants secretly bid 0-10 thrill points
2. **Compare Bids**: Higher bid wins the round
3. **Victory Mechanics**:
   - Bid > Enemy: Opponent loses 1 HP
   - Bid = Enemy: Clash (both lose 1 HP, no winner)
   - Bid < Enemy: Player loses 1 HP
4. **Match Resolution**: First to 0 HP loses; victor gains XP
5. **Leveling**: Victory grants XP; reach thresholds to unlock new levels

#### Progression
```
Level 1: 20 XP to Level 2
Level 2: 40 XP to Level 3
Level 3: 80 XP to Level 4
... (XP requirement doubles each level)
```

#### Daily Limits
- Injuries persist (can't duel if HP = 0)
- HP regenerates at daily reset (each in-game day)

#### Strategic Value
- Tournaments offer prestige and rare rewards
- Champion stats used to determine ties in militia power (edge cases)
- Public leaderboards show top combatants

### Module Breakdown

#### Core Modules
- **constants.js**: Defines all game data (NPC lords, unit stats, building costs, Fabrial effects). Change multipliers/costs here to balance the game.
- **game-state.js**: Manages player state creation, loading (from localStorage), and saving. Handles state migrations for game updates.
- **utils.js**: Logging system, browser notifications, screen flash effects. Used by all modules.
- **validation.js**: Checks whether actions are valid (enough resources, building cap not exceeded, etc.).

#### System Modules
- **military.js**: Calculates army stats (total power, carry capacity, survival%, speed). Handles recruitment.
- **buildings.js**: Manages construction (checks cost, cap, linearity), Gemheart purchases, Fabrial crafting.
- **arena.js**: Champion leveling, duel mechanics (thrill bidding, XP calculation), tournament entry.
- **espionage.js**: Spy operations (scout/heist/sabotage), suspicion tracking, counter-intelligence, decay mechanics.

#### Event Modules
- **plateau-runs.js**: Spawns Chasmfiend events, NPC coalition formation, victory/defeat resolution, reward distribution.
- **deployments.js**: Army deployment missions (send forces to accomplish objectives, return with loot/casualties).

#### UI Modules
- **ui-manager.js**: Renders all game stats, tabs, resource displays. Updates every second (UI_UPDATE_RATE).
- **modal-manager.js**: Handles modal windows (spy operations, deployments, tournament signup, etc.).

## How to Recreate This Game

If you need to rebuild this game or migrate to a new system, follow this reconstruction guide:

### Phase 1: Core Systems (Weeks 1-2)
1. **Time Architecture**
   - Implement real-time clock (1 hour = 1 day)
   - Calculate elapsed time on login; simulate all intermediate ticks
   - Persist `lastTickTime` to detect offline duration

2. **State Management**
   - Create player object with: spheres, gemhearts, military units, buildings, fabrials
   - Implement save/load to persistent storage (localStorage, database, etc.)
   - Version your saves; add migration logic for future updates

3. **Resource Generation**
   - Daily tick: process Markets (100 S/day), Gemheart income (20 S/day each)
   - Apply Fabrial multipliers (Ledgers: 1.5x Market income)
   - Log all resource changes for transparency

### Phase 2: Military System (Week 2-3)
1. **Unit Definition**
   - Create unit type with stats: power, survival%, carry, speed, cost
   - Implement recruitment (check resources, add to inventory)
   - Create army stats calculation (sum power with multipliers from buildings/Shardbearers)

2. **Unit Tracking**
   - Track available vs. deployed units (subtract deployed from available for recruitment UI)
   - Calculate casualties when units return from missions (50-100% survival rate)

3. **Shardbearers (Power Multiplier)**
   - Implement 2.0x multiplier on army power
   - Shardbearers cost 1 Gemheart (link economics: players must choose Shardbearers vs. Fabrials)

### Phase 3: Buildings & Fabrials (Week 3)
1. **Building System**
   - Define building types with: base cost, scaling (linear/exponential), cap, effects
   - Implement construction (deduct cost, increment building count, show completion time)
   - Apply building effects to calculate stats (Training Camp +10% power, Monastery +5% survival)

2. **Fabrial System** (Requires Research Library)
   - Fabrials crafted from Gemhearts (expensive, limited supply)
   - **Heatrial**: 3 GH → 1.5x provision cap
   - **Ledger**: 5 GH → 1.5x Market income (passive scaling)
   - **Gravity Rig**: 7 GH → 2.0x army speed
   - Stacking: Multiple Fabrials stack effects (e.g., 2 Ledgers = 1.5x × 1.5x = 2.25x income)

### Phase 4: Plateau Run System (Week 4)
1. **Event Generation**
   - Random spawn trigger (e.g., every 2-4 hours of active gameplay)
   - Generate event with difficulty tier (Medium/Hard) and enemy power (50-150)
   - Send player alerts (notifications, screen flash, log message)

2. **Coalition Formation**
   - Player chooses units to commit or abstains
   - Simulate NPC lords joining automatically (random selection from available NPCs)
   - Calculate speed for each participant: `1.0 + (bridgecrews * 0.01)`

3. **Resolution**
   - Check: Total allied power ≥ enemy power?
   - If victory: Fastest participant wins Gemheart; others split consolation pool (power×carry-weighted distribution)
   - If defeat: Casualties applied (5-50% unit loss), no rewards
   - Record results in event log

4. **Rewards Distribution**
   - **Gemheart Winner**: +1 Gemheart + 5,000 Sphere finder's fee
   - **Consolation Players**: Proportional share of 50,000 Sphere pool (weight = power × carry)
   - **No Carry**: Default 5% of consolation pool

### Phase 5: Espionage System (Week 5)
1. **Rival Tracking**
   - Each rival's state: suspicion (0-100), suspicionLevel (unknown/known/suspicious/hostile), spies caught counter
   - Initialize on first interaction

2. **Spy Operations**
   - Implement action functions (scout, spy, heist, sabotage)
   - Check player's available spy power vs. action cost
   - Calculate success: `base_chance × suspicion_multiplier`
   - On success: Grant rewards (intel, Sphere steal, etc.) + increase suspicion by action cost
   - On failure: Spy discovered; suspicion +50, unit casualty possible

3. **Suspicion Decay** (Daily)
   - If no spy action on target this day: Apply exponential decay
   - Day 1 without action: -1 suspicion
   - Day 2: -2 suspicion
   - Day 3: -4 suspicion (doubles each day)
   - Encourages rotating targets rather than spamming one rival

4. **Counter-Intelligence**
   - Rivals with high suspicion generate counter-intel
   - Counter-intel reduces espionage success rate (40% at "Suspicious", 10% at "Hostile")
   - Retaliation events on repeated hostile activities

### Phase 6: Arena Combat (Week 6)
1. **Champion System**
   - Player starts with Champion (Level 1, 3 HP, 10 thrill)
   - XP gain from duel victories: +10 XP per round won
   - Level-up thresholds: `20 × 2^(level-2)` (exponential scaling)
   - On level: +1 max HP, +1 max thrill, spell effects

2. **Duel Mechanics**
   - Both combatants bid 0-10 thrill points
   - Compare bids; higher bid deals 1 damage to opponent
   - If equal: Both take 1 damage (risky!)
   - Match ends when one combatant reaches 0 HP
   - Winner gets XP; loser stays at 0 until next day

3. **Daily Limits**
   - Max 5 duels per in-game day
   - All stats reset at daily tick (HP restored, thrill pool refreshed)

### Phase 7: UI & Polish (Week 7)
1. **Tab System**: Military, Buildings, Arena, Espionage, Plateau Runs
2. **Real-Time Updates**: UI refreshes every 1 second (UI_UPDATE_RATE)
3. **Notifications**: Browser notifications + in-game log
4. **Modals**: Deployment, recruitment, spy operations, tournament signup
5. **Leaderboards**: Top players by level, power, suspicion

## Running the Game

The game requires a **local HTTP server** to serve the files (due to ES module loading restrictions).

### Using Node.js (Recommended)
```bash
npm install -g http-server
cd "path/to/Warcamps  Gemini 2.0"
http-server -p 8000
```

Then open: `http://localhost:8000/Warcamps.html`

### Using Python
```bash
cd "path/to/Warcamps  Gemini 2.0"
python3 -m http.server 8000
```

Then open: `http://localhost:8000/Warcamps.html`

## Dev Mode (Fast Testing)

Dev mode allows you to test game systems quickly by accelerating time and providing starting resources.

### How to Enable Dev Mode

1. Open `assets/js/core/constants.js`
2. Change line 2 from `export const DEV_MODE = false;` to `export const DEV_MODE = true;`
3. Refresh the game in your browser

### Dev Mode Features

When enabled, you'll get:
- ⚡ **10-second days** instead of 1 hour (test 24 days in 4 minutes!)
- 💰 **25,000 Spheres** starting resources (vs 15,000)
- 💎 **10 Gemhearts** starting resources (vs 0)
- ⚠️ **Yellow console message** confirming dev mode is active

### What to Test

With these settings, you can quickly verify:
- **Plateau Runs**: Spawn in ~30 seconds, resolve quickly
- **Arena Duels**: Fight multiple times and level up your champion
- **Building Construction**: Buy expensive buildings instantly
- **Fabrial Engineering**: Craft all fabrials without grinding
- **Daily Events**: See the full event cycle in minutes

### Disable Dev Mode

Change `DEV_MODE = true` back to `false` in `assets/js/core/constants.js` for normal gameplay.
